# Method Overview

## Goal

The goal is to convert EEG responses into visual-language captions. Instead of forcing EEG directly into an LLM hidden space, this code first maps EEG to structured visual semantics, then lets a language model compose a caption from retrieved evidence.

## 1. EEG Feature Extraction

The EEG encoder is based on EEGPT. The pretrained EEGPT backbone is loaded from a local checkpoint and kept frozen. A lightweight trainable adapter maps EEGPT features into the project feature space.

Why: the dataset is too small to train a strong EEG encoder from scratch. Freezing EEGPT keeps general EEG representation ability, while the adapter specializes it for this task.

## 2. Coarse Object Classification

Stage 1 trains the adapter/classifier to predict image object labels from EEG.

Why: object category is the most stable and useful cue for captioning. Even when the final sentence is generated by LLM, a coarse object prior helps restrict retrieval candidates and reduce hallucination.

## 3. Structured Caption Targets

Qwen-VL is used offline to generate one structured English sentence per image. The prompt asks for category, action/state, visual attributes, and scene/context. The generated text is normalized to make the format consistent.

Why: original captions can be short, inconsistent, or missing low/mid/high information. Structured captions make semantic supervision cleaner.

## 4. Low / Mid / High Semantic Database

Each image receives three semantic texts:

- low semantic: color, texture, shape, size, local visual attributes
- mid semantic: main object, object state, action, spatial layout
- high semantic: scene, context, event-level or global meaning

The texts are encoded by CLIP text encoder and stored as semantic retrieval targets.

Why: the three branches are not three different labels. They are three descriptions of the same image from different semantic levels.

## 5. Multi-Head Semantic Alignment

Stage 2 trains a `MultiHead` module:

```text
EEGPT feature
  -> low head  -> low CLIP text embedding
  -> mid head  -> mid CLIP text embedding
  -> high head -> high CLIP text embedding
```

Losses:

- low contrastive loss
- mid contrastive loss
- high contrastive loss
- full-caption contrastive loss on the high branch
- auxiliary classification cross entropy

Why: contrastive loss teaches each EEG semantic branch to retrieve the matching image semantics from a batch, rather than merely minimizing pointwise MSE.

## 6. Retrieval and Evidence Decision

At inference, each EEG sample predicts low/mid/high semantic embeddings. Each branch retrieves top-k semantic anchors from the database. The evidence decision step combines:

- branch retrieval score
- branch importance
- classifier top-k candidates
- agreement among retrieved anchors

The result is a compact set of reliable cues and uncertain cues.

Why: EEG is noisy. Directly trusting one predicted label often fails. Evidence aggregation makes the output more conservative and more explainable.

## 7. LLM Caption Generation

The final prompt contains reliable semantic cues, uncertain cues, and a strict instruction to generate one concise sentence. The structured prompt version is tuned to match reference-caption style, improving lexical metrics such as F1, BLEU, ROUGE, and METEOR.

Why: the LLM is used as a controlled composer, not as an unconstrained guesser.

## 8. Evaluation

The evaluation script reports:

- token F1
- CLIP text similarity
- optional Sentence-BERT similarity if a local model is provided
- BLEU-1/2/3/4
- corpus BLEU-4
- ROUGE-L
- METEOR-like score
- CIDEr-like score

These metrics cover lexical overlap and semantic similarity.
